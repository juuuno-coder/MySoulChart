# OKR 가이드 & 바이브 철학관 OKR

---

## Part 1: OKR 작성 가이드

### OKR이란 무엇이며 왜 하는가?

OKR은 도달하고 싶은 **Objective(목표)**와 이를 달성했는지 확인할 수 있는 **Key Results(핵심 결과)**로 구성됩니다. 많은 기업이 이를 도입하지만 실패하는 이유는 단순히 '측정 가능한 수치'만 나열하거나, 리더가 일방적으로 할당하기 때문입니다.

**핵심**: OKR은 '관리용' 도구가 아니라, 구성원들이 목표에 집중하고 협업하게 만드는 **'정렬(Alignment)' 도구**입니다.

---

### 1단계: Objective (도전적 목표)

"우리가 도달하고 싶은 곳은 어디인가?"에 대한 답.

- 가슴 뛰고 영감을 주는 **정성적인 문구**로 작성할 것
- 단순히 "기능 구현"이 아니라 **"사용자에게 주는 가치"**에 집중할 것

| 구분 | 예시 |
|------|------|
| Bad | "이번 분기 매출 20억 달성" (단순 수치는 가슴을 뛰게 하지 못함) |
| Good | "모든 애견인이 좋아하는 브랜드로 도약한다" (가치 중심적, 방향성 명확) |

---

### 2단계: Key Results (핵심 결과)

"목표 달성 여부를 어떻게 알 수 있는가?"에 대한 구체적인 지표.

- **수치로 측정 가능**해야 함 (수익, 사용자 수, 고객 만족도 등)
- **공격적이고 도전적**이어야 함 (약 70% 정도 달성 가능한 수준)
- 최소 **3가지 지표**를 제안할 것

| 구분 | 예시 |
|------|------|
| Bad | "고객 응대 시스템 안정화" |
| Good | "시스템 오류 0%, 구매 전환율 40% 달성" |

---

### 3단계: Initiatives (실행 방안)

- KR을 달성하기 위해 **당장 코드 레벨에서 실행**해야 할 구체적인 작업(To-do)을 나열할 것

---

### 핵심 원리: Output이 아닌 Outcome에 집중하라

가장 흔한 실수는 Key Results 자리에 '해야 할 일(Output/Task)'을 적는 것.

| 구분 | 잘못된 예 (Output) | 올바른 예 (Outcome) |
|------|-------------------|-------------------|
| KR | "새로운 디자인 시안 3개를 만든다" | "디자인 변경을 통해 구매 전환율을 30% 높인다" |
| KR | "버그 100개 수정" | "서비스 안정성 지표 99.9% 달성" |

---

### 핵심 요약

> - **Objective**: 우리가 도달하고 싶은 원대한 꿈 (가슴 뛰는 목표)
> - **Key Results**: 그 꿈에 다가갔음을 증명하는 결과적 수치 (측정 가능한 지표)
> - **Initiative**: 이를 위해 우리가 실제로 실행하는 행동 (할 일 목록)

---

### 실전 사례

**Objective**: "더 많은 구매 전환을 일으키는 높은 품질의 웹사이트를 런칭한다"

**Key Results**:
- KR 1: 방문자 지속시간 100% 상승
- KR 2: 구매 전환율 30% 상승

**Initiatives**:
- [ ] 새로운 디자인 시안 3개 제작
- [ ] 웹사이트 개발자 2명 신규 채용

---

### 결론

OKR은 단순히 작성법을 익힌다고 끝나는 것이 아닙니다. "이 목표를 왜 해야 하는가?"를 이해하고, 스스로 도전적인 지표를 설정할 때 비로소 강력한 힘을 발휘합니다.

## Part 2: 바이브 철학관 OKR (2026 Q1)

### 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | **바이브 철학관** (Vibe Philosophy Agent 3.0) |
| 핵심 기능 | 사주, 관상, MBTI, 혈액형을 통합한 AI 철학 상담 서비스 |
| 기술 스택 | React 19 + TypeScript + Vite 6 + Gemini 2.0 Flash |
| 분석 모드 | 통합 심층 / 커플 궁합 / 관상학 / 혈액형 심리 / MBTI 인지구조 / 사주명리 (6가지) |
| 최종 목적 | 자신을 탐구하고 싶은 사용자에게 전통과 현대가 결합된 깊이 있는 자기이해 경험 제공 |

---

### Objective

> **"사용자가 자신의 내면을 깊이 이해하고, 삶의 방향성을 찾을 수 있는 가장 신뢰받는 AI 철학관 플랫폼이 된다"**

단순한 운세 서비스가 아닌, 사용자가 자신의 본질을 발견하고 실질적인 인사이트를 얻어 인생의 중요한 결정을 내릴 때 가장 먼저 찾는 플랫폼.

---

### Key Results

#### KR 1: 사용자 몰입도 극대화

| 항목 | 내용 |
|------|------|
| 목표 | 평균 세션 시간 **15분 이상** 달성 |
| 현재 | 측정 시스템 구축 필요 (베이스라인 미정) |
| 측정 방법 | Google Analytics 4 세션 타임 추적 |
| 달성 의미 | 사용자가 단순 호기심이 아닌, 진지한 대화를 나누는 수준 |

#### KR 2: AI 응답 만족도 향상

| 항목 | 내용 |
|------|------|
| 목표 | 사용자 피드백에서 "도움이 되었다" 평가 **85% 이상** |
| 현재 | 만족도 설문 시스템 구축 필요 |
| 측정 방법 | 대화 종료 시 만족도 설문 (5점 만점 중 4점 이상 비율) |
| 달성 의미 | AI 응답이 피상적이지 않고 실질적인 통찰을 제공 |

#### KR 3: 재방문율 (Retention) 향상

| 항목 | 내용 |
|------|------|
| 목표 | 30일 내 재방문율 **40% 이상** 달성 |
| 현재 | 세션 영속성 구현 완료 (LocalStorage 24시간 유효) |
| 측정 방법 | GA4 Retention 리포트 + LocalStorage 기반 추적 |
| 달성 의미 | 고민이 생겼을 때 다시 찾아오는 서비스 |

#### KR 4: 서비스 안정성 및 응답 속도

| 항목 | 내용 |
|------|------|
| 목표 | 평균 AI 응답 시간 **3초 이내**, 에러율 **1% 미만** |
| 현재 | 에러 핸들링 개선 완료 (Toast 시스템, 친화적 에러 메시지) |
| 측정 방법 | Gemini API 응답 시간 로깅 + 에러 로깅 |
| 달성 의미 | 사용자가 기다림 없이 자연스러운 대화 흐름 경험 |

---

### Initiatives (실행 계획)

#### Phase 1: 핵심 안정화 (완료)

- [x] Toast 알림 시스템 구축 (alert() 전면 교체)
- [x] 파일 업로드 검증 (5MB 제한, MIME 타입 확인)
- [x] 데이터 영속성 (LocalStorage 세션 저장/복원)
- [x] 세션 복원 모달 (이어하기/새로 시작 선택)
- [x] 모바일 드로어 (세션 중 ControlPanel 접근)
- [x] API 에러 메시지 친화적 개선 (429, 403, 404 대응)
- [x] API 키 보안 (console.log 노출 제거)

#### Phase 2: UX 향상 (진행 예정)

- [ ] 온보딩 모달: 첫 방문자를 위한 인터랙티브 서비스 소개
- [ ] 대화 내보내기: JSON/텍스트/이미지 형식 다운로드
- [ ] 이미지 미리보기: 관상 사진 업로드 후 썸네일 + 분석 결과 요약 표시
- [ ] 접근성 개선: ARIA 속성, 키보드 네비게이션, 색상 대비 (WCAG AA)

#### Phase 3: 측정 및 고도화 (향후)

- [ ] Google Analytics 4 연동: 세션 시간, 모드 전환, 메시지 전송 이벤트 추적
- [ ] 만족도 설문 시스템: 세션 종료 시 5점 척도 + 자유 의견 수집
- [ ] 성능 최적화: 메시지 히스토리 슬라이딩 윈도우, React.memo/useMemo
- [ ] 스켈레톤 UI: 로딩 상태 시각적 개선
- [ ] Tailwind CSS npm 마이그레이션: CDN에서 로컬 빌드로 전환
- [ ] A/B 테스팅: 프롬프트 및 UI 변경 효과 측정

#### 성장 기반 (Phase 3 이후)

- [ ] SNS 공유 기능: 상담 결과를 카드 이미지로 공유
- [ ] SEO 최적화: 메타 태그, 구조화된 데이터
- [ ] 랜딩 페이지: 서비스 소개 및 가치 전달

### OKR 자기 평가 (Self-Correction)

**이 OKR이 '할 일 목록'이 아닌 '비즈니스 가치 창출'에 집중하는가?**

| 검증 항목 | 결과 |
|----------|------|
| Key Results가 Outcome(결과) 중심인가? | O - 세션 시간, 만족도, 재방문율, 응답 속도 모두 사용자 경험 결과 |
| 측정 가능한 구체적 수치가 있는가? | O - 15분, 85%, 40%, 3초/1% 등 명확한 수치 |
| 70% 달성 시에도 성공이라 느낄 만큼 도전적인가? | O - 15분 세션, 85% 만족도는 상당히 높은 목표 |
| Initiatives가 KR 달성에 직접 기여하는가? | O - 각 Phase가 특정 KR을 지원 |

**개선 필요 사항**:
- 현재 베이스라인 데이터(세션 시간, 재방문율 등)가 없으므로 **Phase 3에서 GA4 연동 후 초기 측정 필수**
- 만족도 설문 시스템이 구축되어야 KR 2 측정 가능
